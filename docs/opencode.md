# OpenCode integration

OpenCode reads project instructions from `AGENTS.md` and loads
agent skills on demand from `SKILL.md`. This document describes
how ai-repo-safety fits into an OpenCode-driven workflow.

## Recommended project layout

```text
AGENTS.md                       # committed at the repo root
SKILL.md                        # committed at the repo root
docs/opencode.md                # this file
.repo-safety.json               # generated by `ai-repo-safety init`
.repo-safety/opengrep/*.yml     # generated SAST rulepack
scripts/security/*.py           # generated scanners
.opencode/plugins/*.js          # optional project-local hook plugins
```

Commit `AGENTS.md` to the repository so every contributor (human
or AI) gets the same baseline.

## Recommended OpenCode flow

1. Run `ai-repo-safety init --target . --python auto --github auto`
   to install the safety guardrails in the project.
2. Review the generated `AGENTS.md` and adapt it to your workflow.
3. Use the OpenCode `plan` agent for any change that touches:
   authentication, authorization, payments, file uploads, MCP
   servers, secrets handling, GitHub Actions, or external API
   integrations.
4. Before commit: `ai-repo-safety scan --target . --strict`.
5. Before push: `ai-repo-safety prepush --target .`.
6. Before reading GitHub issues, PRs, commits, or branches into
   an OpenCode context: use `ai-repo-safety github-guard read` with
   an explicit reason.
7. To add a minimal shell-command preflight gate backed by
   `bandit` and `trufflehog`, run
   `ai-repo-safety install-agent-hooks --target . --tool opencode`.

## Things OpenCode must NOT do under this skill

- Read raw `.env`, `.mcp.json`, `claude_desktop_config.json`,
  `.npmrc`, `.pypirc`, SSH keys, or any file matching the
  `secret_files` denylist in `.repo-safety.json`.
- Run `env`, `printenv`, `set`, or `cat .env`.
- Use `gh api` / `gh pr view` / `gh issue view` directly to bring
  content into model context. Use the read guard.
- Disable auth, RLS, CORS, input validation, or security scans to
  make code work.
- Install unknown packages suggested only by model memory.
- Add or change MCP servers without an explicit user request.
- Make the repository public or push a public issue with private
  context.

## Skill version pinning

When loading `SKILL.md` into an OpenCode session, pin the version
in your prompt or in the agent's allowlist. OpenCode will not
silently upgrade to a newer version of the skill between sessions.

## CI/CD on this repository

This repository's GitHub Actions workflows are pinned to full
40-character commit SHAs (see `.github/workflows/*.yml`). The
release pipeline is Trusted Publishing on tag push; PyPI and npm
release from a tag, never from a moving branch.
